In diesem Workshop wurden Sie in die Amazon Pinpoint SMS- und Voice-V2-API eingeführt und haben die Grundlagen von SMS kennengelernt. Durch eine Reihe praktischer Übungen mit der AWS CLI und den V2-APIs haben Sie Pinpoint-SMS-Komponenten eingerichtet, Testnachrichten gesendet, CloudWatch-Protokolle aktiviert und überprüft. Sie haben einen SMS-Wiederholungsmechanismus implementiert und CloudFormation verwendet, um eine vorgefertigte Lösung zum Streamen, Speichern und Abfragen von SMS-Ereignissen bereitzustellen.

Um mit SMS zu beginnen, müssen Sie im Voraus wissen, in welchen Ländern Sie SMS senden möchten und welcher geschätzte [Durchsatz](https://docs.aws.amazon.com/sms-voice/latest/userguide/sms-limitations-mps.html) erforderlich ist. Verwenden Sie diese Daten, um die [Ursprungsidentität zu identifizieren](https://docs.aws.amazon.com/pinpoint/latest/userguide/channels-sms-countries.html)，die Ihren Anforderungen entspricht, und planen Sie etwas Zeit ein, insbesondere wenn einige Länder eine Registrierung erfordern. Nachfolgend finden Sie eine Liste empfohlener Schritte für die erfolgreiche Einrichtung Ihres SMS-Programms mit Amazon Pinpoint:

1. Beantragen Sie [Kurzcodes](https://docs.aws.amazon.com/sms-voice/latest/userguide/phone-numbers-request-short-code.html)，[Langcodes](https://docs.aws.amazon.com/sms-voice/latest/userguide/phone-numbers-long-code.html)，[Absender-IDs](https://docs.aws.amazon.com/sms-voice/latest/userguide/sender-id-request.html) und schließen Sie gegebenenfalls die Registrierung ab. Weitere Informationen zu Registrierungen finden Sie [hier](https://docs.aws.amazon.com/sms-voice/latest/userguide/registrations.html).
2. [Verschieben Sie Amazon Pinpoint SMS in die Produktionsumgebung](https://docs.aws.amazon.com/sms-voice/latest/userguide/sandbox.html#sandbox-sms-move-to-production)
3. [Beantragen Sie eine Erhöhung des SMS-Ausgabenlimits](https://docs.aws.amazon.com/sms-voice/latest/userguide/awssupport-spend-threshold.html). Das monatliche SMS-Ausgabenlimit bestimmt, wie viel Geld Sie pro Monat für das Senden von SMS-Nachrichten über Amazon Pinpoint ausgeben können. Wenn Amazon Pinpoint feststellt, dass das Senden einer SMS-Nachricht die Kosten über Ihr Ausgabenlimit für den aktuellen Monat hinaus erhöhen würde, stoppt es das Veröffentlichen von SMS-Nachrichten innerhalb weniger Minuten.
4. Richten Sie die für Ihren Anwendungsfall erforderlichen SMS-Komponenten ein, wie in [Lab 1](/lab1-sms-setup) gezeigt, einschließlich Pools, Opt-out-Listen, Konfigurationssätzen und Ereigniszielen.
5. Verwenden Sie den [SMS-Simulator](https://docs.aws.amazon.com/sms-voice/latest/userguide/test-phone-numbers.html) von Amazon Pinpoint, um den Durchsatz und die Logik Ihrer Anwendung für die Verarbeitung von SMS-Ereignissen zu testen.
6. Verwenden Sie den [Phone number validate](https://docs.aws.amazon.com/pinpoint/latest/developerguide/validate-phone-numbers.html)-Service von Amazon Pinpoint, um zu bestimmen, ob eine Telefonnummer gültig ist, und um zusätzliche Informationen über die Telefonnummer selbst zu erhalten. Dies kann Ihnen helfen, Kosten zu sparen, indem Sie Festnetzanschlüsse oder ungültige Nummern herausfiltern.
7. Wenn Sie Spitzen in Ihrem SMS-Verkehr erwarten, sollten Sie [Aufrufe an Amazon Pinpoint in eine Warteschlange stellen](https://aws.amazon.com/blogs/messaging-and-targeting/queueing-amazon-pinpoint-api-calls-to-distribute-sms-spikes/). Die Amazon Pinpoint SMS- und Voice-V2-API stellt keine Nachrichten in die Warteschlange, die die zugewiesenen Nachrichtenteile pro Sekunde für ein Land überschreiten. Daher sollte Ihre Anwendung die API-Antworten von Amazon Pinpoint lesen und entsprechend warten oder die Nachrichten in eine Warteschlange stellen.
8. Befolgen Sie die [SMS-Best Practices](https://docs.aws.amazon.com/sms-voice/latest/userguide/best-practices.html).
